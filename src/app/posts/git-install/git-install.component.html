Git LFS (Large File Support)
預設已勾選。
主要是讓 Git 能夠處理大型檔案（10 MB 以上）。因為 Git 的設計上都是以文字檔的程式碼為主，而程式碼的大小通常都不會太大，都是很多小型檔案為主。［by: 哈部落：Git LFS 原理、大小檔案都適用］

Associate .git* configuration files with the default text editor
預設已勾選。
預設使用預設的文字編輯器打開所有檔名為「.git」的檔案（ex: .git, .gitignore, .gitkeep）。

Associate .sh files to be run with Bash
預設已勾選。
預設使用 Bash 打開所有檔名為「.sh」的檔案（Shell Script）。

Use a TrueType font in all console windows
預設已勾選。
命令列視窗是否要支援 Unicode，Win 10 以上的版本已內建支援 Unicode，所以可不勾選。

Check daily for Git for Windows updates
預設未勾選，建議勾選。
每天會檢查 Git for Windows 是否有新版本，且自動更新為新版。

6. 調整 Path 環境變數

選擇要如何加入 Git 相關的環境變數，使用預設值只將 Git 加入環境變數。選擇預設值，如下圖，直接下一步就好。



以下介紹一下圖中三個選項的意思：

Use Git from Git Bash only
不將 Git 執行檔加入環境變數，也就是在命令列視窗無法下「git」執行 Git 相關指令，只可於 Git Bash 中使用 Git 指令。

Use Git from the Windows Command Prompt
預設值（建議選項），將 Git 執行檔加入環境變數，也就是在命令列視窗下「git」即可執行 Git 相關指令。

Use Git and optional Unix tools from the Windows Command Prompt
將 Git 執行檔及 Git Bash 下的指令皆加入環境變數，也就等同於將命令列視窗與 Bash 結合。
建議命令列視窗與 Bash 分開，以明確知道兩者的指令差異。

8. 換行字元（Line Ending Conversions）

選擇使用換行字元的方式，因應跨平台的需求，在不同作業系統下的換行字元不同，需要設定在切換平台時對於換行符號的調整方式。 例如在 Windows 跟 Linux 下使用的換行字元就不一樣，前者使用 CRLF（Carriage Return and Line Feed，「/r/n」）作為換行字元，後者則使用 LF（Line Feed，「/n」）。

此預設值為在讀取檔案時（Checkout），使用本機作業系統的換行方式顯示；在本地儲存庫時（Commit），使用 LF 的換行方式儲存。 建議使用此選項，如下圖，比較不會因換行字元的關係，介面跑掉或是記錄錯誤（例如只是因換行字元不同，而造成變更數增加，導致錯誤的歷史紀錄），直接進入下一步。



其他選項的介紹如下：

Checkout Windows-style, commit Unix-style line endings
預設值（建議選項）。
在讀取檔案時，使用本機作業系統的換行方式顯示；在本地儲存庫時，使用 LF 的換行方式儲存。

Checkout as-is, commit Unix-style line endings
在讀取檔案時，使用該檔案原本的換行方式顯示；在本地儲存庫時，使用 LF 的換行方式儲存。
通常是原本作業系統就使用 LF 為換行字元時，才會選擇此選項。

Checkout as-is, commit as-is
在讀取檔案與儲存到本地儲存庫時，皆使用該檔案原本的換行方式
較不易於跨平台開發。

補充：Will 保哥：Git 在 Windows 平台處理斷行字元 (CRLF) 的注意事項


9. Git Bash 的終端模擬器（Terminal Emulator）

選擇 Git Bash 的執行環境，像是 java 需要運行在 JVM 環境下一樣，Git Bash 也需要其執行環境。使用預設值 MinTTY，如下圖，直接進入下一步即可。



以下簡單介紹一下圖中兩種執行環境：

Use MinTTY (The default terminal of MSYS2)
預設值（建議選項）。
為終端模擬器，模擬 Linux 的環境，但依然能存取 Windows 作業下的檔案，以及支援 Unicode。 缺點是執行 Python 時會有一些問題，細節可以參考圖中的說明。

Use Windows' default console window
命令列視窗的環境，只能使用命令列的指令，細節可以參考圖中的說明。


10. 其他設定

選擇其他要設定的選項，使用預設值「啟用快取、紀錄 Git 驗證」即可，下一步後一樣是附加功能，一樣使用預設值，即不勾選， 最後點選「Install」就可以開始安裝啦～





以下為四項設定的介紹：

Enable file system caching
預設已勾選（建議勾選）。啟用快取，在大型專案開發時可以加快讀取速度。

Enable Git Credential Manager
預設已勾選（建議勾選）。紀錄 Git 驗證，每次要上傳/下載遠端儲存庫的檔案時皆需要登入 Git，開啟這個選項即會紀錄登入資料，就不需每次都重新登入。

Enable symbolic links
預設未勾選。勾選後，Git 只能使用系統管理員身分的命令列視窗才能使用，一般命令列視窗則無法使用 Git。

Enable experimental, built-in add -i/-p
預設未勾選。勾選後，加入新的指令（'git add -i' or 'git add -p'），主要是可以指定上傳檔案中的某一部份，目前還不穩定，建議不勾選，等穩定再說:P
