0. 為何需要版本控管？

在大型專案中非常仰賴版控，因其需區分發行版本與開發版本等多種版本，並需詳細記錄所有的變更，包含內容的變更、變更的人員、變更的時間， 且為每一個時刻原始碼的版本，也需比對不同版本的差異，甚至在必要時，能夠復原檔案的變更等等。

此外，在多人開發時，版本控管也是很重要的一環，能夠精準的紀錄每個人員修改的版本，以及管理、分配每個人負責的區塊。

最後，需要實現 DevOps（Development & Operations），也就是開發、維運的一體化，即仰賴版控的技術才得以實現。


1. 版本控管的種類

要實施版本控管，有三種方式，本地版本控管、集中式版本控管及分散式版本控管。

1.0. 本地版本控管

使用資料夾備份來記錄每一次的版本，也就是最陽春的版本控管。

每當要更新版本、做變更時，就再複製一份資料夾，這使得需儲存大量的檔案，且絕大多數都是重複並未更改的檔案，可見佔位極大。

1.1. 集中式版本控管

使用遠端儲存庫儲存每一次的版本，且只有在遠端伺服器上才有完整的版本庫（儲存庫）。

每當要更新版本、做變更時，就須將變更丟到遠端儲存庫，其他開發者再從遠端儲存庫下載更新的版本。

但，這會產生一個問題，每當開發人員要上傳新版本時，就必須先連到遠端儲存庫去下載目前最新的版本，再將自己修改的新版本加入後，最後上傳到遠端儲存庫， 且要觀看版本的歷史訊息、下載最新版都需要網路連線。

而其優點，就是能輕易地替每個檔案做權限管理，因為完整的檔案只存在遠端儲存庫，所以只要到遠端儲存庫對檔案進行權限設定即可。

常見的集中式版控為 CVS(Concurrent Version System)、Subversion、Team Foundation Version Control、Perforce。下圖為集中式版本控管的示意圖：



1.2. 分散式版本控管

使用遠端儲存庫儲存每一次的版本，且遠端伺服器與本地端皆有完整的版本庫（儲存庫）。

每當要更新版本、做變更時，只需將更動丟到本地儲存庫，需合併不同開發者的更動時，再上傳到遠端儲存庫即可。

查詢版本的歷史訊息不需連網，因為在本地就有完整的歷史紀錄，每次更動也不需到遠端下載，大幅減少傳輸時間，增加效率。

而其缺點，就是難於做檔案的權限管理，因每位開發人員在本地端皆有完整的儲存庫，無法集中標示特定檔案的權限，僅能設定遠端儲存庫上的權限（例如上傳、下載）。

2. Git 分散式版控軟體

Git 是目前廣為人知的版控軟體，是一個分散式版本控制軟體，最初由林納斯·托瓦茲創作，於2005年以GPL釋出，最初目的是為更好地管理Linux核心開發而設計。［by: 維基百科：Git］

Git 提供了版本庫儲存所有的歷史紀錄、版本比較、合併、追蹤、標記等功能，幾乎想地到的管理方式，Git 都幫你準備好了！

其軟體下載方式也非常簡單，以 windows 作業系統為例（本人為 win10），只需於 Git 官網下載即可， 下載時的一些設定與選項介紹，將於下篇章做教學，敬請期待～


3. GitHub 平台

GitHub 是提供遠端儲存庫的平台，是透過 Git 進行版本控制的軟體原始碼代管服務平台，由 GitHub 公司（曾稱Logical Awesome）的開發者 Chris Wanstrath、PJ Hyett 和 Tom Preston-Werner 使用 Ruby on Rails 編寫而成。［by: 維基百科：GitHub］

不過不單只提供遠端儲存庫而已，GitHub 也提供像是 GitHub Page，供使用者上傳文章、靜態網站（0x01 從零開始的個人網站） 其平台使用方式只需於GitHub 官網註冊 GitHub 帳號，即可使用囉～而且 GitHub 也有預設提供給 Git 軟體的指令，便於串聯到 GitHub 遠端儲存庫唷！

因此只要搭配 Git 一同使用，就可以開始使用分散式版本控管啦～

以上就是今天的課程，我們來重點整理一下：

  • 版本控管：針對專案的各個版本做控制、管理
  • 版控的好處：紀錄每一個時刻原始碼的版本、比對不同版本的差異、復原檔案的變更
  • 版本控管的種類：
    • 本地版本控管：使用資料夾備份來記錄每一次的版本
    • 集中式版本控管：使用遠端儲存庫儲存每一次的版本，且只有在遠端伺服器上才有完整的版本庫（儲存庫）
    • 分散式版本控管：使用遠端儲存庫儲存每一次的版本，且遠端伺服器與本地端皆有完整的版本庫（儲存庫）
  • 集中式版控 v.s 分散式版控：
    • 集中式：優點-易於做檔案權限設定；缺點-觀看版本的歷史訊息、下載最新版都需要網路連線
    • 分散式：優點-查詢版本的歷史訊息、更動皆不需連網，在本地就有完整的歷史紀錄；缺點-不易做檔案權限設定
  • Git：分散式版本控制軟體，提供了版本庫儲存所有的歷史紀錄、版本比較、合併、追蹤、標記等功能
  • GitHub：是透過 Git 進行版本控制的軟體原始碼代管服務平台，其中提供遠端儲存庫
