1. Angular 環境建置

Step 1：至官網下載 Node.js 與開發工具（Visual Studio Code）

需使用 Node.js 的工具下載 Angular CLI（Step2 會介紹）唷！

（Node.js是一個能夠在伺服器端運行 JavaScript 的開放原始碼、跨平台 JavaScript 執行環境 ［by: 維基百科: Node.js］）

VSCode（Visual Studio Code）則為開發工具，對於撰寫程式、版控都非常方便！

（也可選擇較擅長的開發工具）

Step 2：下載 Angular CLI

於命令列視窗（cmd）打上下面指令，接著等它跑完就安裝完成了～（遇到詢問的就 yes 到底）

npm install -g @angular/cli

Angular CLI 為命令列工具，可以輕鬆於命令列視窗下指令去創建、新增、操作 Angular 的專案
  • [npm install]：Node.js下載指令
  • [-g]：global的縮寫，為全域下載，本台電腦任何地方皆可使用（如不加，則僅下載的目錄下可使用）
  • [@angular/cli]：要下載的檔案（Angular CLI）
Step 3：於VS Code 下載 Angular Extension Pack（will 保哥）擴充套件

打開 VSCode，在左側選單欄中找到Extensions，進入後，搜尋「Angular Extension Pack」

找到「will 保哥」的套件，點選 install 即可

這個套件自動安裝好這十幾個好用、實用、必用的 Angular 相關擴充套件，快謝保哥！！



完成以上步驟後，就準備好 Angular 的開發環境囉！

2. 建立 Angular 專案 & 部屬

Step 1：建立專案

於命令列視窗（cmd）打上下面指令，接著會詢問你是否要加上 routing 以及使用的裝飾樣式，這裡就先「yes」然後選擇「CSS」吧！

ng new test
  • [ng]：Angular CLI 指令
  • [new]：創建一新專案
  • [test]：專案名稱，可以取你尬意的


Step 2：開啟 Angular 頁面

進入專案中，於命令列視窗打上下面指令，接著打開任一瀏覽器（Chrome），於網址列輸入「localhost:4200」

能看到顯示出Angular初始畫面，就代表Angular專案以建置成功囉！

ng serve
or
npm start
  • [ng serve]：建置當前目錄的 Angular專案，使用全域的 Angular CLI 版本建置
  • [npm start]：執行 ng serve，建置當前目錄的 Angular專案，使用當前專案的 Angular CLI 版本建置（下一篇會介紹）
Step 3：部屬 Angular 專案

進入專案中，於命令列視窗打上下面指令，等它跑好就部屬好了～

預設部屬於專案中的 dist 檔案中（下一篇會詳細介紹部屬完成的檔案）

ng build --prod
  • [build]：部屬當前目錄的 Angular專案，使其可以發行
  • [--prod]：壓縮後部屬，即移除空白字元、空行等。部屬後的檔案的檔名掛上後綴

3. Git、GitHub 版控建置

以後網站可能需要多人一起經營、開發，為了能方便於不同電腦上撰寫我們的網站

所以我們需要共享專案，這時候版控就很重要了！

（現在只簡單介紹 git 的基礎用法，之後還會另開頁面做 git 教學唷！）

Step 1：至官網下載git

安裝就都使用預設值，一路下一步到底就對了！！

git 是一個分散式版本控制軟體，最初目的是為更好地管理 Linux 核心開發而設計。 ［by: 維基百科: Git］

Step 2：至官網註冊GitHub

我們需要一個平台去控管我們的專案，可以看到修改的內容、分支等等，註冊就不多闡述～ GitHub 是透過 Git 進行版本控制的軟體原始碼代管服務平台 ［by: 維基百科: GitHub］

Step 3：將 Angular 專案丟上版控（github）

首先，需於 GitHub 上建立專案（repository），如下圖



輸入好專案名稱（建議與本地端專案名稱相同），可以選擇要公開或是保密，如下圖



上圖藍色框起來的部分，使用預設值即可，即不建立 .gitignore 跟 license

　
  • .gitignore：不加入版控的檔案會列於當中，Angular 專案預設以建立此檔案
  • license：開源軟體授權條款（詳細請看：在開源時代的興起下，如何透過License共享並保有權益）

點選「Create Repository」後就建立好一個 GitHub 專案啦～而如何把現在本地端的專案丟到 GitHub 版控中呢？

由於 Angular 預設以配置好 git 文檔（於隱藏的資料夾 .git），所以只需於 Angular 專案下，執行以下指令就完成了喔～

git config --global user.email "you@example.com"
git remote add origin https://github.com/yourUserName/repositoryName.git
git push -u origin master
　
  • [git]：git 版控工具

  • [config --global user.email "you@example.com"]：

    • [config]：取得和設定 git 的組態參數，這裡是為了登入用

    [user.email "you@example.com"]：使用 email 登入 GitHub，也可使用「user.name "Your Name"」以使用者名稱登入

  • [remote add origin https://github.com/yourUserName/repositoryName.git]：

    • [remote]：指向遠端版本控制倉庫，就是指到版控平台（GitHub）上

    [add origin]：加入遠端版本控制倉庫的路徑，選擇剛剛創建好的 GitHub 專案，讓本地端 Angular 的專案聯繫上

    [https://github.com/yourUserName/repositoryName.git]：即剛剛創建好的 GitHub 專案路徑 （yourUserName：GitHub 註冊時的 username，repositoryName：GitHub 專案名稱）

  • [push]：將本地端檔案丟進版控，即將檔案上傳到分支上（簡單來說就是丟上 GitHub 平台，不過這真的是簡而言之）

  • [-u origin master]：

    • [-u]：相等於「--set-upstream」，即設定 upstream，可以使目前本地端開始追蹤指定的遠端分支

    [origin master]：指定的遠端分支為 master（master 為分支名稱，預設的主要分支為 master，也可自行改名）
可以到 GitHub 上看是不是已把本地端的檔案都丟上專案了呢？很酷吧！
